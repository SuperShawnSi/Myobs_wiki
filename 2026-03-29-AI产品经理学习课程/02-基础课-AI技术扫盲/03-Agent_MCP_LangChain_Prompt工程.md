# Agent、MCP、LangChain 与 Prompt 工程

## 这篇课解决什么问题

这几个词是 AI 面试里最容易被混着说的：
- Agent
- MCP
- LangChain / LangGraph
- Prompt 工程

很多人一股脑都叫“AI 框架”或者“AI 工作流”，这很危险。

这篇课的目标是：

1. 让你知道这四个词根本不是一类东西。
2. 让你知道它们分别解决什么问题。
3. 让你知道什么时候该用，什么时候别乱叫。
4. 让你能把它们和自己的项目连起来。

## 一、先给一个最短对照表

| 名词 | 它是什么 | 它解决什么问题 |
|---|---|---|
| Agent | 一类能围绕目标组织步骤并调工具的系统 | 任务执行 |
| MCP | 一种开放协议 | 标准化连接外部系统 |
| LangChain / LangGraph | 工作流框架 | 编排多步骤 LLM 流程 |
| Prompt 工程 | 输入与指令设计方法 | 让模型更稳定地按要求工作 |

先记住：

- **Prompt** 不是框架
- **MCP** 不是模型
- **LangChain** 不是协议
- **Agent** 不是所有 AI 功能的统称

## 二、什么是 Prompt 工程

我们先从最基础的讲，因为它最容易落地。

### 1. 什么叫 Prompt
Prompt 就是你给模型的输入内容。

但在实际工作里，Prompt 通常不只是一个问题，而是一整套说明：
- 你是谁
- 你要做什么
- 你不能做什么
- 你该按什么格式输出
- 这里有哪些上下文
- 这里有没有示例

### 2. 为什么 Prompt 工程重要
因为模型不是“自动懂你想要什么”。

如果你只说：

```text
帮我写一段描述
```

模型可能输出：
- 风格不统一
- 长短不合适
- 胡乱补充信息
- 格式不可用

但如果你这样写：

```text
你是一个摄影作品元数据助手。
请根据图片内容、EXIF 和已填草稿，生成：
1. 一个 15 字以内的标题
2. 3 到 5 个标签
3. 一段不超过 60 字的描述
不要编造具体地名或人物姓名；如果不确定，就返回空。
输出必须是 JSON。
```

结果就会稳定很多。

### 3. Prompt 工程到底在做什么
它本质上是在做 6 件事：
- 定义角色
- 定义任务
- 定义规则
- 定义格式
- 提供上下文
- 提供示例

### 4. 一个最小 Prompt 模板

```text
角色：你是谁
任务：你要完成什么
输入：我会给你什么信息
规则：哪些能做，哪些不能做
输出：必须按什么结构返回
示例：给你一个参考样例
```

### 5. 对你最重要的理解
Prompt 工程不是“写得花哨”，而是：

**用清晰、稳定、可约束的方式把模型行为压进产品边界里。**

## 三、什么是 Agent

### 1. 先给最简单定义
Agent 不是单次回答，而是：

**围绕一个目标，自主决定步骤，并可能调用工具完成任务的系统。**

OpenAI 对 Agent 的表达很明确：
- 有模型
- 有工具
- 有指令与护栏
- 能代表用户完成任务

### 2. 为什么不是所有 AI 功能都叫 Agent
下面这些通常不算完整 Agent：
- 单次总结
- 单次分类
- 单次改写
- 单次图片描述生成

因为它们更像：
- 一次请求
- 一次输出
- 中间没有复杂决策和工具协同

### 3. 一个更像 Agent 的例子
假设你有一个“差旅助理”：

```text
用户：帮我安排下周去上海两天出差
```

系统可能会：
1. 先解析日期和目的地
2. 查公司差旅政策
3. 查机票
4. 查酒店
5. 比较价格和时间
6. 生成候选方案
7. 等用户确认后再下单

这种：
- 有目标
- 有多个步骤
- 可能调多个工具
- 会根据中间结果调整后续动作

才更像 Agent。

### 4. Agent 的基本组成

```text
Agent = 模型 + 工具 + 指令/护栏 + 状态/流程控制
```

缺一个都可能不完整。

### 5. 你的项目里什么不是 Agent

#### `PP@SS AI 元数据助手`
它更像：
- 多模态输入
- Prompt 约束
- 结构化输出
- 人工确认

但它没有：
- 多步自主决策
- 一串工具调用
- 基于中间状态持续规划

所以它**不是完整 Agent**。

#### `COMP9900`
它更像：
- RAG + 规则设计 + 结果展示

也不是完整 Agent。

这点很重要：

**不要为了显得高级，把所有 AI 功能都叫 Agent。**

## 四、什么是 MCP

### 1. 先给一句人话
MCP 是：

**让 AI 应用标准化连接外部系统的协议。**

如果你觉得这句太抽象，你可以把它理解成：

**给 AI 装一个标准化接口，让它更容易接文件、数据库、搜索、工具和业务系统。**

### 2. 为什么它会出现
如果没有统一标准，接工具通常会变成：
- 每个系统都各写一套接口
- 每个模型应用都自己造一遍轮子
- 接一个数据库是一种方式
- 接一个搜索工具又是另一种方式

这样会很乱。

MCP 的价值就在于：
- 统一约定
- 标准化暴露工具和资源
- 降低对接复杂度

### 3. MCP 里常见的概念
你现在不用学实现细节，但要知道几个词：
- `MCP client`：发起调用的一侧
- `MCP server`：暴露工具/资源的一侧
- `tool`：可执行动作
- `resource`：可读取内容
- `prompt`：可复用提示模板

### 4. 一个很简单的例子
假设一个 AI 助手需要读取公司知识库和数据库。

没有 MCP 时，可能是：
- 每种外部能力都要单独写适配层

有 MCP 时，你可以把它理解成：
- AI 通过统一协议去问：
  - 这里有哪些工具？
  - 我能读哪些资源？
  - 我能调什么动作？

### 5. 对 AIPM 来说，MCP 该懂到什么程度
你不需要自己实现协议，但你要知道：
- MCP 解决的是连接问题，不是模型能力问题
- MCP 不是 Agent
- MCP 不是 RAG 的替代品
- MCP 更像一层标准化接入能力

## 五、什么是 LangChain / LangGraph

### 1. 它们不是模型，也不是协议
LangChain / LangGraph 更像是：

**组织 LLM 工作流的框架。**

### 2. 你什么时候会需要它们
如果你只是：
- 单次问答
- 单次生成
- 单次分类

那未必需要它们。

如果你做的是：
- 多步骤任务
- 状态流转
- 工具路由
- 分支判断
- 复杂编排

那它们就会变得更有价值。

### 3. LangChain 和 LangGraph 粗暴怎么区分

#### LangChain
更像一套组件库和工作流搭建工具。

#### LangGraph
更像面向状态机和多节点流程的编排方式。

你现在不需要深入 API，但要知道：

**它们解决的是“流程怎么组织”，不是“模型本身怎么变强”。**

## 六、这四个东西之间的关系

很多人听完会继续问：它们到底怎么串起来？

你可以这样理解：

### Prompt 工程
解决：
- 怎么把模型说明白

### LangChain / LangGraph
解决：
- 多步骤流程怎么组织

### MCP
解决：
- 工具和外部系统怎么标准化接入

### Agent
解决：
- 如何围绕一个目标，结合模型、工具和流程去完成任务

### 一个组合例子

```text
用户提出任务
  -> Agent 决定先做什么后做什么
  -> 工作流用 LangGraph 组织
  -> 需要调用知识库和工具时，通过 MCP 接入
  -> 每一步里，模型按 Prompt 工程定义的规则工作
```

## 七、代码层面怎么理解

你不需要写完整框架，但可以看一个极简伪代码。

### 4. 一个更像 Agent 的逻辑

```python
goal = "帮用户完成差旅安排"

while not done:
    step = llm.plan_next_step(goal, current_state)

    if step.type == "search_flights":
        result = flight_tool.search(step.params)
    elif step.type == "search_hotels":
        result = hotel_tool.search(step.params)
    elif step.type == "ask_user":
        result = ask_user(step.question)

    current_state.update(result)
```

这里你能明显感觉到：
- 它不再是单次问答
- 而是围绕目标持续规划和执行

## 八、你在面试里怎么回答这几个词

## 1. Agent 是什么
```text
我理解 Agent 不是简单套了大模型的聊天框，而是围绕一个目标，能组织步骤、调用工具并完成任务的系统。它通常至少包含模型、工具和行为规则，有时还需要状态管理和失败回退。
```

## 2. MCP 是什么
```text
MCP 是让 AI 应用标准化连接外部系统的开放协议。它更像一层标准化接入能力，解决的是工具和资源怎么统一暴露给 AI，而不是直接提升模型本身的能力。
```

## 3. LangChain / LangGraph 是什么
```text
它们更像组织 LLM 工作流的框架。重点不在于模型更强，而在于多步骤流程、状态和工具调用怎么编排。
```

## 4. Prompt 工程是什么
```text
Prompt 工程不是写几句提示词，而是通过角色、规则、上下文、示例和输出格式，把模型行为约束到产品要求里，让输出更稳定、更可控。
```

## 九、初学者最容易犯的 6 个错误

### 错误 1：把所有 AI 功能都叫 Agent
错。

### 错误 2：把 MCP 理解成某个模型能力
错。它是协议。

### 错误 3：把 LangChain 理解成“必须要用的东西”
错。简单任务未必需要。

### 错误 4：把 Prompt 工程理解成“写得越长越高级”
错。重点是清晰、稳定、可约束。

### 错误 5：认为用了 Agent 就自然更高级
错。很多功能用普通 workflow 就够了。

### 错误 6：只背定义，不会拿项目举例
面试最怕这个。
