# 先学什么，后学什么

## 如果你时间只有 3 天

只看这些：
1. `AI 产品经理到底做什么`
2. `PRD / 原型图 / 用户故事 / 验收标准`
3. `Transformer / RAG / Agent / MCP / Prompt 工程`
4. `如何把你的项目讲成 AIPM 能力`
5. `AIPM 高频面试题与答题框架`

## 如果你时间有 7 天

按这个顺序：
1. 产品经理底座全部读完
2. AI 技术扫盲全部读完
3. 提高课中的 `如何把AI能力做成产品` 与 `如何设计AI结果边界`
4. 冲刺课全部读完

## 如果你时间有 14-30 天

按完整顺序走，并同步做三件事：
1. 每读一篇，写自己的项目映射
2. 每学一块，补一次简历或讲稿
3. 每周至少做一轮模拟面试

## 学习时的三个提醒

### 1. 不要把“术语理解”误当成“能做产品”
知道 `RAG`、`Agent`、`Prompt 工程` 还不够。你必须能回答：
- 为什么这里要用它？
- 不用它会怎样？
- 它的风险是什么？
- 怎么评估它是否有效？

### 2. 不要只学 PM 词汇，不学 AI 边界
传统 PM 很多问题靠流程就能解。AI PM 不是。AI PM 的关键在于：
- 能力边界
- 错误边界
- 成本边界
- 安全边界
- 人工兜底

### 3. 不要脱离自己的项目学
你的优势从来不是“会背标准答案”，而是：
- `COMP9900` 里已经做过需求拆解、故事板、验收标准、迭代推进
- `PP@SS` 里已经做过 Dashboard、AI 元数据助手、访问控制、稳定性治理
- `9321 / 9444 / 9414 / 9517` 已经给你补了数据与模型理解

所以你不是从 0 开始，而是在把零散经验系统化。
