# 面试复盘表与错题本

## 为什么你需要这份文档

很多人面完试只记得“感觉一般”“好像答得不太好”，这种复盘没有价值。

真正有效的复盘要回答三件事：
1. 面试官到底问了什么
2. 我哪里答得不够好
3. 下次我要改哪一句、补哪块知识

## 一、面试后 10 分钟内先记什么

每场面试结束后，立即记下：
- 公司和岗位
- 面试轮次
- 问到的项目
- 问到的技术点
- 你卡住的地方
- 你觉得答得最好的一题

## 二、复盘表模板

你可以按下面格式写：

```text
公司 / 岗位：
日期：
轮次：

一、问了哪些题
1.
2.
3.

二、我答得好的地方
1.
2.

三、我答崩的地方
1.
2.

四、问题属于哪一类
- 产品基础
- AI 技术理解
- 项目深挖
- 协作与行为面
- 岗位匹配

五、下次怎么改
- 要补什么知识
- 要换哪种讲法
- 要把哪个项目讲得更清楚
```

## 三、错题本怎么分类

### 1. 概念不会
例如：
- MCP 到底是什么
- Prompt 工程和工作流有什么区别

### 2. 项目讲不清
例如：
- 为什么 `PP@SS AI 元数据助手` 不是 Agent
- `COMP9900` 到底交付了什么

### 3. 回答太虚
例如：
- 一直说“体现了能力”
- 没有具体交付和结果

### 4. 情绪和节奏问题
例如：
- 紧张后开始狂堆术语
- 说话太快
- 回答太长

## 四、你现在最该重点记录的题

### AIPM 岗位通用
- 你理解的 AI 产品经理是什么
- 为什么你适合做 AI PM
- 你没有实习，为什么还要考虑你

### 项目题
- `COMP9900` 为什么像 AI 产品
- `PP@SS AI 元数据助手` 为什么不是 Agent
- 你怎么设计人工确认和失败兜底

### 技术理解题
- Transformer 是什么
- RAG 是什么
- Agent / MCP / Prompt 工程分别是什么
- 你怎么评估一个 AI 功能

## 五、如何把复盘转成改进动作

每场面试后，不要只说“下次更自信一点”。

你要改成这种动作：
- 补读哪篇文档
- 重写哪个问题的答案
- 重录哪段项目讲稿
- 下次先讲什么再讲什么

### 示例
问题：我把 RAG 和 Agent 讲混了。
改进动作：
- 重读 `03-Agent_MCP_LangChain_Prompt工程.md`
- 用 100 字写一版“RAG / Agent / MCP 的差异”
- 明天口述 3 次

## 六、你该如何判断自己有没有进步

不是看“我今天学了多久”，而是看：
- 以前卡住的问题，这次能不能说顺
- 以前很虚的答案，这次能不能落到项目
- 以前容易混淆的概念，这次能不能区分

## 七、最低要求

从现在开始，每场模拟面试和真实面试，至少留下：
1. 一份复盘表
2. 三个错题
3. 一个重写后的答案

这样你后面准备面试，才会越来越快，而不是每次都从头来过。
