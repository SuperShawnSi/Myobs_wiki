# 面试问题应对包

这套资料不是泛泛的“产品经理面试题库”，而是专门为你当前这类候选人准备的：

- 具备前端与一定全栈背景
- 正在从技术背景转向 `AI 产品经理`
- 已有 `COMP9900`、`PP@SS`、`9321 / 9444 / 9414 / 9517` 这些项目与课程证据
- 需要应对中国大陆 AI 产品经理 / 产品经理 / 用户产品经理的真实面试

## 这套包解决什么问题

它主要解决五类问题：

1. 你知道很多概念，但不知道面试官到底会怎么问。
2. 你有不少项目，但不会把它们讲成产品语言。
3. 你能说术语，但不会把术语和需求、指标、交付边界连起来。
4. 你担心自己没有实习，会被追问“为什么适合做 AI PM”。
5. 你需要把公开面经里的高频问题沉淀成可反复训练的固定知识网。

## 目录结构

- `00-开始这里`：总说明、设计逻辑、使用方法
- `01-自我介绍与转岗动机`：解决“你是谁、为什么转 AIPM、为什么不是前端主线”
- `02-简历逐条深挖`：逐条拆你的主投简历，反推问题和答法
- `03-AI产品经理专业知识`：把 AI PM 常见专业问题组织成系统题库
- `04-产品设计与开放题`：0 到 1 设计题、Case 题、AI 场景改造题
- `05-数据指标与评估`：埋点、指标、漏斗、Dashboard、实验与评估
- `06-行为面与压力追问`：协作、冲突、短板、无实习、真实性质疑
- `07-中国大陆高频真题索引`：把公开面经里的常见问题统一收口
- `08-按项目索引-COMP9900_PP@SS_9321等`：按你的项目快速找题和答法

## 这一轮先落什么

这轮不直接把所有主文档写满，而是先完成：

1. 目录骨架
2. 大陆高频真题总表
3. 外部来源映射
4. 每个分册一份“本册说明与待写范围”

这样做的原因是：
- 先把问题空间收束
- 再按优先级逐册扩写
- 避免后面出现重复题、同题多版、目录返工

## 推荐使用顺序

1. 先读 [00-开始这里/00-本册说明与待写范围.md](./00-开始这里/00-本册说明与待写范围.md)
2. 再读 [00-开始这里/01-中国大陆高频问题总表.md](./00-开始这里/01-中国大陆高频问题总表.md)
3. 然后按以下优先级扩写与训练：
   - `01-自我介绍与转岗动机`
   - `02-简历逐条深挖`
   - `03-AI产品经理专业知识`
   - `05-数据指标与评估`
4. 最后再补 `04-产品设计与开放题` 和 `06-行为面与压力追问`

## 写作标准

后续每个主文档扩写时，都要满足以下标准：
- 用中国大陆真实面试语境写，不写空泛北美职场套话
- 题目不是只列问题，而要解释“为什么问”
- 答案不是只给方向，而要能落到你的项目和证据链上
- 必须兼顾 HR 扫读、业务面、产品面和技术追问
- 文风、密度和结构尽量与课程现有主文档保持一致
